# 📌 Background & Business Problem

## Business Context

VietDist is a nationwide FMCG distribution company operating through:

* Sales representatives
* Customers
* Distributors
* Promotion programs
* Sales targets

Business stakeholders need a centralized reporting layer to answer:

* How much revenue is generated by each employee?
* Are employees achieving their assigned sales targets?
* Which distributors perform best?
* What products generate the highest revenue?
* How do returns impact business performance?

The raw operational data is scattered across multiple CSV and Excel files and cannot be used directly for reporting.

---

# 🎯 Project Objective

Build an end-to-end Data Warehouse using Bronze-Silver-Gold architecture.

The final solution should:

* Centralize all business data
* Standardize and clean raw datasets
* Support historical target versioning
* Build Star Schema for analytical reporting
* Produce business-ready data marts for Power BI

# 🥉 Bronze Layer – Data Ingestion

## Objective

Load raw source files into PostgreSQL while preserving original data.

### Key Activities

* Built Python ingestion pipeline
* Loaded CSV and Excel sources
* Added ingestion metadata

### Metadata Columns

* _source_file
* _source_platform
* _ingested_at
* _batch_id

### Special Case: Sales Target Versioning

The sales target file contains multiple planning versions:

* Plan_v1_Original
* Plan_v2_Adjustment_H2

All versions are preserved in Bronze without overwriting historical plans.

## Sales Target Versioning Logic

Implemented business logic for target planning:

| Period     | Effective Version |
| ---------- | ----------------- |
| Month 1–6  | v1                |
| Month 7–12 | v2                |

Additional fields:

* effective_from
* effective_to
* is_latest

This allows historical tracking of planning adjustments throughout the year.
